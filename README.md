## üå∏ Overview

The CycleVault is a **privacy-first menstrual cycle tracking app** designed around a single principle:

> **Your reproductive health data should remain fully under your control.**

Unlike most health apps, The CycleVault does not require accounts, does not sync data to the cloud, and does not rely on analytics or advertising SDKs. All core functionality works entirely offline, with data stored locally on your device and protected by iOS security mechanisms.

The project intentionally prioritizes **clarity, restraint, and trust** over optimization, gamification, or behavioral tracking.

---

## üé® Core Philosophy

The CycleVault is built with a small set of non-negotiable values:

- **Local-first by default**  
  All cycle data stays on your device. Nothing is uploaded automatically.

- **Explicit user control**  
  Any external connection only happens when a user deliberately chooses it.

- **No monetization through data**  
  No ads, no analytics SDKs, no silent telemetry.

- **Clarity over prediction**  
  The app emphasizes awareness and patterns, not guarantees or ‚Äúcycle hacking.‚Äù

- **Calm, human-centered design**  
  The interface and content are intentionally gentle, neutral, and non-alarmist.

The visual language (warm coral, soft lavender, minimal contrast) reflects this philosophy: approachable, steady, and respectful.

## üåô What the App Does

The CycleVault allows users to:

- Log period start and end dates
- Track flow intensity, symptoms, and notes
- View cycles on a calendar and in history
- See estimated cycle phases and patterns
- Export or fully delete all data at any time
- Lock the app using Face ID / Touch ID (optional)
- Read menstrual health education written in plain English

All features are designed to work **without creating an account** and **without sharing personal information**.

## üîí Privacy & Transparency

Privacy is a product feature, not a promise.

The CycleVault intentionally avoids:
- Analytics SDKs
- Crash reporting tied to identity
- Background network activity
- Third-party data collection libraries

A built-in **Privacy Dashboard** allows users to see:
- What data exists on their device
- How much data is stored
- When the app last made an external connection (if ever)

This transparency is designed to be observable, not assumed.
